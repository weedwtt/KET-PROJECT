# KET-PROJECT — Product Context

## Product

ระบบบันทึกและติดตามความประพฤตินักเรียน สำหรับโรงเรียนบางพลีราษฎร์บำรุง ใช้งานภายในสถาบันการศึกษา ครอบคลุมกระบวนการตั้งแต่บันทึกเหตุการณ์ผิดวินัย จนถึงการอนุมัติหลายชั้น และการส่งออกเอกสารราชการ (PDF / Word)

ระบบแบ่งเป็น 2 ประเภทหลัก:
- **บันทึกข้อความ (Statement)** — รายงานเหตุการณ์ผิดวินัยของนักเรียนรายบุคคล
- **สัญญาข้อตกลง (Bond)** — สัญญาความประพฤติสำหรับนักเรียนที่มีประวัติซ้ำ

## Users

| บทบาท | หน้าที่ |
|---|---|
| ครูผู้บันทึก (Recorder) | กรอกข้อมูลเหตุการณ์และรายละเอียดนักเรียน |
| ครูปกครอง (Discipline Teacher) | ตรวจสอบและลงนามอนุมัติขั้นแรก |
| หัวหน้าระดับ (Grade Head) | อนุมัติขั้นที่สอง |
| รองผู้อำนวยการ / ผู้อำนวยการ | อนุมัติสุดท้าย และมีสิทธิ์เต็มระบบ |

ผู้ใช้ส่วนใหญ่ไม่ใช่ผู้เชี่ยวชาญด้านเทคโนโลยี ใช้คอมพิวเตอร์เพื่องานราชการ ต้องการ UI ที่เข้าใจง่าย ตรงไปตรงมา

## Brand & Tone

- **ภาษา:** ไทยเป็นหลัก label, ข้อความ, และ error message ทั้งหมดต้องเป็นภาษาไทย
- **บุคลิก:** เป็นทางการ, น่าเชื่อถือ, ตรงไปตรงมา — ไม่ใช่ startup, ไม่ใช่ social app
- **ความรู้สึก:** ระบบราชการที่ใช้งานง่าย ดูมืออาชีพ ไม่ฉูดฉาด
- **สี:** ใช้สีโทนเย็น (น้ำเงิน, เทา) เน้นความน่าเชื่อถือและความสะอาด

## Core Flows

1. **บันทึกความประพฤติ** → กรอกข้อมูลนักเรียน, เลือกประเภทความผิด, ระบุมาตรการ
2. **Approval chain** → ครูปกครอง → หัวหน้าระดับ → ผู้อำนวยการ (พร้อมลายเซ็น)
3. **ส่งออกเอกสาร** → พิมพ์เป็น PDF หรือ Word ตามฟอร์มราชการ
4. **รายงาน** → dashboard สถิติการกระทำผิด, trends รายปีการศึกษา

## Anti-References

- ไม่ต้องการให้ดูเหมือน social media (ไม่มี card feed, ไม่มี avatar prominent)
- ไม่ต้องการ animation หนัก, glassmorphism, gradient text
- ไม่ต้องการ dark mode เป็น default (ระบบราชการใช้หน้าจอ light)
- ไม่ต้องการ layout ที่ซับซ้อน — ผู้ใช้ต้องการเจอข้อมูลที่ต้องการทันที

## Strategic Principles

- **Clarity first:** label ทุกอย่างชัดเจน ไม่ให้ผู้ใช้ต้องเดา
- **Trust through form:** เอกสารที่ export ต้องดูเป็นทางการและถูกต้องตามแบบฟอร์มโรงเรียน
- **Zero tolerance for data loss:** form ต้องมี validation เข้มงวด ไม่ให้บันทึกข้อมูลไม่ครบ
- **Role-aware UI:** แสดง action ที่เหมาะสมตาม role ของผู้ใช้ที่ login อยู่
